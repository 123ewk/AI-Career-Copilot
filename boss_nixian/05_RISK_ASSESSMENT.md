# Boss 直聘 API 拦截方案风险评估

## 1. 置信度评估

| 评估项 | 结论 | 说明 |
|--------|------|------|
| **API 端点识别** | 高 | 通过网络抓包直接确认 |
| **请求参数分析** | 高 | 参数含义明确，分页机制简单 |
| **响应结构解析** | 高 | JSON 结构清晰，字段含义明确 |
| **拦截方案可行性** | 高 | 标准的 fetch/XHR 拦截，无特殊防护 |

## 2. 稳定性预期

### API 稳定性

| 风险因素 | 风险等级 | 说明 |
|----------|----------|------|
| 端点 URL 变更 | **低** | `/wapi/zpgeek/pc/recommend/job/list.json` 是核心 API |
| 参数格式变更 | **低** | page/pageSize/city 是标准分页参数 |
| 响应结构变更 | **中** | 字段可能新增或重命名，但核心字段稳定 |
| 认证机制变更 | **中** | 可能增加签名校验 |

### 预期稳定期

- **短期（1-3个月）**: 高度稳定，无需修改
- **中期（3-6个月）**: 可能有小幅调整，需关注字段变化
- **长期（6个月+）**: 建议定期验证 API 可用性

## 3. 使用建议

### 推荐方案：拦截复用

**强烈推荐**通过拦截页面原生请求来获取数据，而非自行构造请求。

#### 优势

1. **无需处理认证**: 自动复用页面的登录态
2. **无需处理签名**: 如有签名机制，自动绕过
3. **实时性好**: 与页面数据同步
4. **维护成本低**: API 变更时只需调整解析逻辑

#### 劣势

1. **依赖页面加载**: 必须在 Boss 直聘页面运行
2. **无法主动触发**: 只能被动等待页面发起请求

### 不推荐方案：自行构造请求

#### 原因

1. **需要登录态**: Cookie 会过期，需要频繁刷新
2. **可能有签名**: 未来可能增加签名校验
3. **IP 限制**: 频繁请求可能触发风控
4. **法律风险**: 可能违反服务条款

## 4. 安全注意事项

### 必须遵守

1. **不硬编码凭证**: Cookie、Token 等敏感信息不得写入代码
2. **不绕过安全机制**: 不破解签名、不伪造请求头
3. **不批量爬取**: 仅用于个人浏览器扩展
4. **不对外服务**: 不提供 API 代理服务

### 建议措施

1. **请求频率限制**: 建议每分钟不超过 10 次请求
2. **错误处理**: 捕获失败时优雅降级
3. **数据缓存**: 避免重复请求相同数据
4. **日志记录**: 记录拦截情况便于调试

## 5. 技术实现要点

### Content Script 隔离

```typescript
// Manifest V3 配置
{
  "content_scripts": [
    {
      "matches": ["*://www.zhipin.com/web/geek/jobs*"],
      "js": ["content.js"],
      "world": "ISOLATED",  // Content Script 运行在隔离世界
      "run_at": "document_start"
    }
  ]
}
```

### 主世界注入

```typescript
// Content Script 中注入主世界脚本
chrome.scripting.executeScript({
  target: { tabId: tab.id },
  world: 'MAIN',
  files: ['interceptor.js'],
});
```

### 消息传递流程

```
主世界拦截器
    ↓ window.postMessage
Content Script
    ↓ chrome.runtime.sendMessage
Service Worker
    ↓ 存储/处理
扩展 UI
```

## 6. 待确认问题清单

| 序号 | 问题 | 验证方法 | 优先级 |
|------|------|----------|--------|
| 1 | 搜索接口是否使用相同结构 | 搜索关键词后抓包 | 高 |
| 2 | 分页请求是否有频率限制 | 连续翻页测试 | 中 |
| 3 | 登录态过期后的错误响应 | 等待 Cookie 过期后观察 | 中 |
| 4 | 是否有 WebSocket 推送 | 检查网络面板 | 低 |
| 5 | 不同城市的 API 行为是否一致 | 切换城市测试 | 低 |

## 7. 后续优化建议

### 功能扩展

1. **搜索接口支持**: 捕获搜索结果页的 API
2. **职位详情获取**: 利用 securityId 获取完整 JD
3. **批量导出**: 支持导出为 CSV/JSON
4. **智能筛选**: 基于薪资、地点、经验等筛选

### 性能优化

1. **增量更新**: 只更新新增的职位
2. **去重机制**: 基于 jobId 去重
3. **懒加载**: 按需加载职位详情
4. **本地缓存**: 使用 IndexedDB 缓存数据

### 用户体验

1. **状态指示**: 显示拦截状态和数据量
2. **手动触发**: 支持手动刷新数据
3. **通知提醒**: 新职位匹配时通知
4. **数据可视化**: 薪资分布、行业分布等
